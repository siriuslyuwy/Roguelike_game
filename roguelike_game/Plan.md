项目计划（几何大战）

目标
- 以 pygame 实现一款 7 战线对推玩法的小型即时战术游戏；可本地运行、键盘操作与简单 AI。

范围（MVP）
1) 4 种单位（圆/方/三角/六边形），含近战与远程两类
2) 资源系统、生产与冷却/费用判定
3) 战线推进、碰撞/对战与基地受击
4) 简单 AI（固定策略或基于压力的随机）
5) 基础 UI：战线、血量、资源、当前选线与键位提示

架构与模块
- main.py：主循环、输入分发、调用 Game 与 UI
- game/constants.py：尺寸、颜色、单位与经济常量
- game/entities.py：UnitType/Unit/Projectile/Base 数据与行为
- game/game.py：Game 核心状态推进、AI 决策、胜负判断
- game/ui.py：绘制战线/单位/投射物/HUD 文案

里程碑
- M1：项目脚手架与依赖（0.5 天）
- M2：数据结构与基础物理（单位推进、同线交战）（0.5 天）
- M3：远程投射物与冷却/射程（0.5 天）
- M4：资源、生产、AI（0.5 天）
- M5：UI/HUD 与交互细节（0.5 天）

数值（初始）
- 资源：初始 100，增速 12/秒，上限 999
- 基地生命：1500 HP
- 单位：
  - 圆：费 25，HP 60，速 110，伤 12，近战，CD 0.6，程 28
  - 方：费 60，HP 220，速 70，伤 20，近战，CD 0.9，程 28
  - 三角：费 70，HP 120，速 90，伤 18，远程，CD 0.9，程 240，弹速 380
  - 六边：费 110，HP 210，速 85，伤 34，远程，CD 1.2，程 280，弹速 420

风险与对策
- 帧率不稳：限制实体数量、使用整数 lane 逻辑、减少碰撞对比
- 输入冲突：固定 1-7 选线，QWER 造兵
- 美术简陋：使用几何形状渲染，后续可替换贴图

测试要点
- 单位对撞与远程射击在同一战线正确发生
- 资源扣减与不足提示不造兵
- 胜负可触发并可 R 重开
