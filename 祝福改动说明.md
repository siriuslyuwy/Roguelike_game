## 改动目标

## 1. 不屈意志 (veteran_unyielding)

### 改动前的问题
- **通关率**: 0% (32个样本)
- **平均层数**: 4.438
- **核心问题**: 
  - 只给了"变强的权限"，但没有给"变强的资源"
  - 其他兵种锻造成功率减半的debuff在前期高压环境下过于致命
  - 起步太慢，前3层容易暴毙

### 改动内容

#### A. 描述文本更新 (`constants.py`)
```python
# 旧描述
"desc": "解锁老兵的锻造与升级权限；代价：其余伙伴锻造成功率减半"

# 新描述
"desc": "开局战士升至2级并获得1攻1防锻造；解锁老兵的商店购买与锻造权限"
```

#### B. 新增开局奖励 (`sim_run.py` + `main.py`)
选择该祝福后立即生效：
- 战士等级：Lv.1 → Lv.2
- 攻击锻造：0 → 1
- 防御锻造：0 → 1

**实现位置**：
- `sim_run.py`: `_maybe_pick_blessing()` 函数
- `main.py`: 祝福选择确认处

#### C. 移除负面Debuff (`main.py`)
删除了"其他兵种锻造成功率减半"的惩罚机制

**删除代码**：
```python
# 已删除
if campaign_run.blessing_selected == "veteran_unyielding" and target_unit and target_unit != "Q":
    p *= 0.5
```

### 预期效果
- 开局2级战士配合锻造，前3层存活率大幅提升
- 移除debuff后，其他兵种的养成不再受阻
- 保持了"战士专精"的特色，但不再是"孤注一掷"

## 2. 教官光环 (veteran_mentor)

### 改动前的问题（第一版改动）
- **通关率**: 0% (29个样本)
- **平均层数**: 4.862
- **核心问题**:
  - Bot不懂得利用"战士后方单位+20%攻速"的核心机制
  - 阵型混乱，经常出现多个坦克堆在一起的情况
  - 没有优先购买战士升级和远程单位

### 改动前的问题（第二版改动）
- **通关率**: 0% (30个样本) - 第一版改动后仍为0%
- **平均层数**: 4.867 - 几乎无变化
- **核心问题诊断**:
  - ❌ **战士伤害-50%太致命**: 变成移动靶子，完全没有输出能力
  - ❌ **推进速度极慢**: 前排推不动 → 后排无法前进 → 攻速buff无法发挥
  - ❌ **数值设计失衡**: 只增加肉度不够，需要保留一定输出能力

### 改动内容（第二版 - 2026-01-14 22:40）

#### A. 数值大幅调整

**战士属性调整**：
```python
# 旧版（失败）
战士 HP +100%, 伤害 -50%  # → HP=320, 伤害=10（太弱！）

# 新版（平衡）
战士 HP +100%, 伤害 -25%  # → HP=320, 伤害=15（保留输出能力）
```

**光环效果增强**：
```python
# 旧版
后方单位: +20% 攻速

# 新版
后方单位: +15% 攻速 + 15% 伤害（双重增益）
```

**设计理念**：
- ✅ 战士保持一定输出能力（伤害15 vs 原版20，减少25%而非50%）
- ✅ 光环提供双重增益（攻速+伤害），补偿战士输出降低
- ✅ 整体DPS提升：战士75% + 后排115% = 整体输出更高

#### B. Bot出兵逻辑优化 (`sim_run.py`) - 第一版改动保留

**核心策略：坦克替换规则**

在正常出兵逻辑的最后，添加替换机制：
```python
# 如果Bot选中的是坦克单位（非战士），且战士可用
# 则自动替换为战士
if blessing == "veteran_mentor" and pick and pick != "Q":
    pick_ut = UNIT_TYPES.get(pick)
    if pick_ut:
        is_tank_unit = (
            not pick_ut.is_ranged 
            and not pick_ut.is_healer 
            and not getattr(pick_ut, "is_buffer", False)
            and float(getattr(pick_ut, "damage", 0)) > 0
            and pick_ut.hp >= 200
        )
        if is_tank_unit and "Q" in game.player_order_keys and game.can_spawn("left", lane, "Q"):
            pick = "Q"  # 替换为战士
```

**优势**：
- ✅ 复用现有的出兵逻辑（已经过500局验证）
- ✅ 不破坏原有的应对机制（AOE、克制等）
- ✅ 自然形成"每条线战士打头"的阵型
- ✅ 战士CD中时，会用其他坦克应急，不会死等

#### B. 商店购买倾向调整 (`sim_run.py`)

在商店的`score()`函数中添加优先级调整：

```python
if blessing == "veteran_mentor" and t == "unit":
    uk = str(it.get("payload") or "")
    ut = UNIT_TYPES.get(uk)
    cur = self._current_unit_level(uk)
    
    # 战士升级：高优先级（8000/6000分）
    if uk == "Q":
        if cur == 1:  # Lv1→2 是质变
            return 8000.0
        elif cur == 2:  # Lv2→3 锦上添花
            return 6000.0
    
    # 远程/DPS单位：次优先级（7000分起）
    if ut and (ut.is_ranged or "dps" in getattr(ut, "tags", [])):
        return 7000.0 - (cur * 500)
    
    # 其他坦克：降低优先级（3000分）
    if ut and "tank" in getattr(ut, "tags", []) and uk != "Q":
        return 3000.0
```

**购买优先级排序**：
1. 战士升级（8000分）
2. 远程/DPS单位（7000分）
3. 技能（5000分，原有逻辑）
4. 其他坦克（3000分）

### 预期效果（第二版）

**理想阵型（后期）**：
```
战线1: [战士Lv3 HP=320 伤害=15] → [弓手Lv2 +15%攻速+15%伤害] → [法师Lv2 +15%攻速+15%伤害]
战线2: [战士Lv2 HP=320 伤害=15] → [冰弓Lv2 +15%攻速+15%伤害] → [牧师Lv1 +15%攻速+15%伤害]
战线3: [战士Lv2 HP=320 伤害=15] → [刺客Lv2 +15%攻速+15%伤害] → [鼓手Lv1 +15%攻速+15%伤害]
```

**DPS计算示例**（以弓手为例）：
```
基础弓手: 伤害25, 攻速1.0 → DPS = 25
光环加成: 伤害25×1.15=28.75, 攻速1.0×1.15=1.15 → DPS = 33.06 (+32%!)
```

**关键指标**：
- ✅ 每条线都有且仅有 **1个战士** 作为前排（Bot逻辑保证）
- ✅ 战士保持输出能力（伤害15 vs 原版20，仍可推进）
- ✅ 后排单位获得 **32%+ 的DPS提升**（攻速×伤害双重加成）
- ✅ 商店优先升级战士到 Lv2/3（确保肉度）
- ✅ 预期通关率：**10-15%**（对比破釜沉舟52%）

## 设计理念

### 不屈意志
**从"赌博型"改为"基石型"**
- 旧版：给权限，但不给资源，还加debuff → 赌博
- 新版：开局直接给资源，移除debuff → 稳健成长

### 教官光环
**从"手动阵型"改为"自动优化"**
- 旧版：需要玩家手动调整站位，Bot完全不会玩
- 新版：Bot自动将坦克替换为战士，自然形成理想阵型
